# 网络协议岗位要求速记

对应招聘要求里的这三条：

```text
3.1 基本了解 TCP/IP 协议，能快速描述 TCP 握手、挥手流程，熟悉 FTP、HTTP、邮件等常用应用协议
3.2 了解以太网，熟练划分 IP 网络，了解静态路由、主机路由、直接路由等概念
3.3 了解常用应用协议，HTTP、FTP、DNS 等
```

## 一、TCP/IP 是什么

TCP/IP 不是单独一个协议，而是一组网络协议。

可以简单理解成：

> 让不同电脑、服务器、手机之间能在网络里互相通信的一套规则。

常见分层：

| 层级 | 作用 | 常见协议 |
|---|---|---|
| 应用层 | 面向应用程序 | HTTP、FTP、DNS、SMTP |
| 传输层 | 负责端到端传输 | TCP、UDP |
| 网络层 | 负责寻址和路由 | IP、ICMP |
| 数据链路层 | 负责局域网内传输 | Ethernet 以太网 |

面试说法：

> TCP/IP 可以理解为互联网通信协议族。应用层有 HTTP、FTP、DNS，传输层有 TCP 和 UDP，网络层主要是 IP，数据链路层常见的是以太网。

## 二、TCP 三次握手

作用：建立可靠连接。

流程：

```text
1. 客户端 -> 服务端：SYN
2. 服务端 -> 客户端：SYN + ACK
3. 客户端 -> 服务端：ACK
```

每一步含义：

| 步骤 | 谁发给谁 | 含义 |
|---|---|---|
| 第一次 | 客户端发 SYN | 客户端请求建立连接 |
| 第二次 | 服务端回 SYN + ACK | 服务端同意连接，并确认收到了客户端请求 |
| 第三次 | 客户端回 ACK | 客户端确认收到了服务端回应 |

为什么要三次：

> 为了确认双方的发送能力和接收能力都正常。

面试说法：

> TCP 三次握手是客户端先发 SYN，服务端回 SYN+ACK，客户端再回 ACK。核心目的是建立连接，并确认双方收发能力正常。

## 三、TCP 四次挥手

作用：断开连接。

流程：

```text
1. 一方发送 FIN
2. 另一方回复 ACK
3. 另一方再发送 FIN
4. 最开始的一方回复 ACK
```

为什么是四次：

> TCP 是全双工通信，双方都可以独立发送数据。所以关闭时，要分别关闭两个方向的数据传输。

面试说法：

> TCP 四次挥手是 FIN、ACK、FIN、ACK。因为 TCP 是全双工连接，双方发送方向要分别关闭，所以通常需要四次。

## 四、TCP 和 UDP 区别

| 对比 | TCP | UDP |
|---|---|---|
| 是否连接 | 面向连接 | 无连接 |
| 是否可靠 | 可靠 | 不保证可靠 |
| 是否有重传 | 有 | 没有内置重传 |
| 顺序保证 | 保证顺序 | 不保证顺序 |
| 速度 | 相对慢 | 相对快 |
| 适合场景 | 网页、文件传输、登录 | 直播、语音、游戏、DNS |

面试说法：

> TCP 面向连接、可靠、有确认和重传机制，适合 HTTP、文件传输等场景。UDP 无连接，不保证可靠，但开销小、速度快，适合实时音视频、游戏、DNS 等场景。

## 五、以太网是什么

以太网英文是 Ethernet。

它主要工作在数据链路层，用在局域网里。

可以简单理解成：

> 同一个局域网里，设备之间怎么传数据的一套规则。

以太网里常见概念：

| 概念 | 意思 |
|---|---|
| 网卡 | 电脑连接网络的硬件接口 |
| MAC 地址 | 网卡的物理地址 |
| 交换机 | 局域网内转发数据的设备 |
| 以太网帧 | 以太网传输数据的基本格式 |

IP 地址和 MAC 地址区别：

| 对比 | IP 地址 | MAC 地址 |
|---|---|---|
| 层级 | 网络层 | 数据链路层 |
| 作用 | 标识网络中的主机位置 | 标识网卡硬件 |
| 是否会变 | 可能变 | 通常固定 |
| 类比 | 收货地址 | 身份证/设备编号 |

面试说法：

> 以太网是局域网中最常见的数据链路层技术，主要通过 MAC 地址在局域网内识别设备，交换机根据 MAC 地址转发以太网帧。IP 更偏网络层寻址，MAC 更偏局域网内实际转发。

## 六、IP、子网掩码、网关

### 1. IP 地址

IP 地址用来标识网络中的一台主机。

比如：

```text
192.168.1.10
```

### 2. 子网掩码

子网掩码用来判断哪些 IP 属于同一个网段。

比如：

```text
IP：192.168.1.10
掩码：255.255.255.0
```

这个网段通常是：

```text
192.168.1.0/24
```

可用主机大概是：

```text
192.168.1.1 到 192.168.1.254
```

其中：

```text
192.168.1.0   通常表示网络地址
192.168.1.255 通常表示广播地址
```

### 3. 网关

网关是访问其他网段时要经过的出口。

比如你的电脑是：

```text
192.168.1.10
```

网关是：

```text
192.168.1.1
```

当你访问百度、公司服务器、其他网段机器时，数据通常会先发给网关，再由网关转发出去。

面试说法：

> IP 标识主机地址，子网掩码用来判断是否在同一网段，网关是访问其他网段时的数据出口。

## 七、怎么划分 IP 网络

重点记住 `/24`、`/25`、`/26`、`/27`。

| CIDR | 子网掩码 | 每个子网地址数 | 可用主机数 |
|---|---|---:|---:|
| `/24` | 255.255.255.0 | 256 | 254 |
| `/25` | 255.255.255.128 | 128 | 126 |
| `/26` | 255.255.255.192 | 64 | 62 |
| `/27` | 255.255.255.224 | 32 | 30 |
| `/28` | 255.255.255.240 | 16 | 14 |

为什么可用主机数要减 2：

```text
一个地址用作网络地址
一个地址用作广播地址
```

所以：

```text
可用主机数 = 地址总数 - 2
```

例子：

```text
192.168.1.0/24
```

如果划分成两个 `/25`：

```text
192.168.1.0/25
范围：192.168.1.0 - 192.168.1.127
可用：192.168.1.1 - 192.168.1.126

192.168.1.128/25
范围：192.168.1.128 - 192.168.1.255
可用：192.168.1.129 - 192.168.1.254
```

面试说法：

> IP 网络划分主要看子网掩码或 CIDR 位数。比如 `/24` 有 256 个地址，可用 254 个；如果划成 `/25`，就分成两个子网，每个子网 128 个地址，可用 126 个。可用地址要减去网络地址和广播地址。

## 八、路由是什么

路由就是：

> 决定数据包下一步往哪里走。

电脑或服务器发包时，会查路由表。

Linux 常见查看命令：

```bash
ip route
```

可能看到：

```text
default via 192.168.1.1 dev eth0
192.168.1.0/24 dev eth0
```

意思：

| 路由 | 含义 |
|---|---|
| `default via 192.168.1.1` | 默认路由，不知道去哪就交给 192.168.1.1 |
| `192.168.1.0/24 dev eth0` | 到 192.168.1.0/24 这个网段，直接从 eth0 出去 |

## 九、静态路由、主机路由、直接路由

### 1. 直接路由

直接路由也叫直连路由。

意思是：

> 目标地址和本机在同一个网段，不需要经过其他路由器，直接从对应网卡发出去。

例子：

```text
本机 IP：192.168.1.10/24
目标 IP：192.168.1.20
```

因为都在：

```text
192.168.1.0/24
```

所以这是直连路由。

### 2. 静态路由

静态路由是管理员手动配置的路由。

意思是：

> 明确告诉系统，去某个网段要走哪个网关。

例子：

```bash
ip route add 10.0.0.0/24 via 192.168.1.1
```

意思：

```text
访问 10.0.0.0/24 这个网段时，走 192.168.1.1 这个网关。
```

### 3. 主机路由

主机路由是只针对某一台主机的路由。

通常是 `/32`。

例子：

```bash
ip route add 10.0.0.5/32 via 192.168.1.1
```

意思：

```text
只访问 10.0.0.5 这一台机器时，走 192.168.1.1。
```

区别总结：

| 类型 | 作用范围 | 例子 |
|---|---|---|
| 直接路由 | 同一网段 | `192.168.1.0/24 dev eth0` |
| 静态路由 | 手动指定某个网段怎么走 | `10.0.0.0/24 via 192.168.1.1` |
| 主机路由 | 只指定某一台主机怎么走 | `10.0.0.5/32 via 192.168.1.1` |

面试说法：

> 直接路由是本机直连网段自动生成的路由；静态路由是人工配置的去某个网段的路径；主机路由是粒度更细的路由，只针对某一个 IP，通常是 `/32`。

## 十、常用应用协议

| 协议 | 全称 | 作用 | 常见端口 |
|---|---|---|---|
| HTTP | HyperText Transfer Protocol | 网页/接口访问 | 80 |
| HTTPS | HTTP Secure | 加密网页/接口访问 | 443 |
| FTP | File Transfer Protocol | 文件上传下载 | 20/21 |
| DNS | Domain Name System | 域名解析成 IP | 53 |
| SMTP | Simple Mail Transfer Protocol | 发送邮件 | 25 |
| POP3 | Post Office Protocol 3 | 收邮件 | 110 |
| IMAP | Internet Message Access Protocol | 收邮件/同步邮件 | 143 |

面试说法：

> HTTP/HTTPS 主要用于网页和接口访问，FTP 用于文件传输，DNS 用于域名解析，SMTP 用于发送邮件，POP3/IMAP 用于接收邮件。

## 十一、最终背诵版

> 我理解 TCP/IP 是互联网通信协议族，应用层有 HTTP、FTP、DNS、邮件协议，传输层有 TCP/UDP，网络层主要是 IP，数据链路层常见的是以太网。TCP 是可靠传输，建立连接是三次握手，断开连接是四次挥手。以太网主要用于局域网内通信，通过 MAC 地址和交换机转发以太网帧。IP 网络划分主要看子网掩码或 CIDR，比如 `/24` 可用 254 个主机地址，`/25` 可用 126 个。路由是决定数据包下一跳，直接路由是同网段直连，静态路由是手动指定去某个网段的路径，主机路由是只针对某一个 IP 的路由。
