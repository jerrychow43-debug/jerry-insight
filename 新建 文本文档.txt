# 🛡️ Jerry-Insight Pro v3.5

### 工业级消费风控与排雷 Agent 引擎（自研 Harness 有限状态机版）

本项目针对传统大模型 RAG 应用在“工具调度死板”、“长文本记忆丢失”以及“输出格式不稳定”等工业级三大缺陷，抛弃了传统的 LangChain 现成框架，手写了一套 **Harness 有限状态机（FSM）控制器**。系统打通了全网去重比价爬虫、混搭向量库、多路消息强推（微信+钉钉）以及带有高并发互斥锁的资产记账闭环。

---

## 🚀 核心技术亮点与工业级痛点解决

| 工业级 Agent 缺陷/痛点 | 本项目硬核技术解决方案（像素级对齐） |
| :--- | :--- |
| **1. 传统 RAG 指代不明（缺陷）** | **前置意图消解路由 (`core.router`)**：在并行检索前，结合短期记忆（Session State）进行模糊词翻译，自动将“它”、“这个”补全为精确商品名词。 |
| **2. 工具调度死板无法闭环（缺陷1）** | **手写 Harness 有限状态机 (FSM)**：基于 `While` 循环构建 Think-Act-Observe 闭环。在大脑发现初始情报不足时，主动下发网络搜索工具进行二轮追查，并将结果逆向喂回大模型，直到触发 `Final Answer`。 |
| **3. 记忆管理简单粗暴截断（缺陷2）** | **自动化窗口摘要压缩机制 (`_compress_memory`)**：短期记忆触达临界点时，自动调用大模型执行流式脱水压缩，生成100字财务核心红线摘要挂载于 System 头部，确保核心规则长期不丢失。 |
| **4. 输出格式断裂导致瘫痪（缺陷3）** | **三层健壮降级容错自愈链**：采用 `try-except` 捕获异常，一二层通过提示词追加进行实时纠偏自愈，最终极限层通过纯文本安全熔断 Fallback 强截获价格块，保证系统 0 崩溃。 |
| **5. 多线程/高并发资产数据错乱** | **操作系统级互斥锁 (`threading.Lock`)**：在对账户余额进行读写改写时，强制拉满 **Mutex Lock（线程安全锁）**，解决并发环境下的资产竞态条件，杜绝多扣/漏扣。 |
| **6. 思考链路黑盒不可见** | **可观测性 Trace 状态机执行树**：引入 LLM Observability 探针，在前端流式渲染出全链路思考树看板，直观追踪大模型每一次内部流转的真实现状。 |

                                      │
                                      ▼
                      [JerryAgentHarness 有限状态机]
                        ├── Think: 格式与合规性审查
                        ├── Act: 自主下发工具二轮追查
                        └── Observe: 核心财务红线比对
                                      │
                                      ▼
                      [三层降级自愈链 / Mutex Lock 线程安全锁]
                                      │
                                      ├──> 前端：Trace 状态机思考树看板 (Streamlit)
                                      └──> 后端：微信推送 + 钉钉群机器人双路通知
