省钱智探 Agent 项目术语速记版

一句话总述：
我做的是一个个人消费决策 Agent。用户输入想买的商品后，系统会做意图分流、商品提取、历史记忆检索、Web Search、价格爬虫和 LLM 审计，最后给出购买建议、估算价格，并支持确认购入、直接记账、撤销、余额加回和钉钉通知。

1. Agent
是什么：LLM + 工具 + 记忆 + 状态 + 流程控制。
项目里：问价时调用 RAG、Web Search、爬虫和 LLM 审计；记账和撤销走规则。
面试说：我不是做普通聊天，而是做能调用工具并维护账本状态的消费决策 Agent。

2. Streamlit
是什么：用 Python 快速做网页应用的框架。
项目里：做聊天界面、侧边栏、按钮、历史记录、账本和 Trace 展示。
面试说：Streamlit 适合快速搭建 AI 原型，我用它完成了项目主版本。

3. Python
是什么：项目主要开发语言。
项目里：写 Agent 逻辑、Streamlit 页面、搜索爬虫、通知、账本和评测脚本。
面试说：Python 负责后端逻辑、工具调用和数据处理。

4. 商品问价
是什么：用户问某个商品值不值得买。
项目里：提取商品名，搜索价格和评价，再让 LLM 给建议。
面试说：商品问价是主入口，后面会进入完整 Agent 审计链路。

5. 实时比价
是什么：获取当前外部价格线索。
项目里：用 Tavily 搜索 + 什么值得买价格爬虫。
面试说：实时比价让 Agent 不只依赖模型旧知识。

6. API
是什么：程序之间通信的接口。
项目里：调用 Tavily API 获取搜索结果，调用 DeepSeek API 获取 LLM 回答。
面试说：API 让我的程序能请求外部 AI 和搜索服务。

7. Tavily
是什么：面向 LLM / Agent 的搜索 API。
项目里：根据商品关键词返回网页摘要、链接和相关性分数。
面试说：Tavily 是我的 Web Search 工具，用来给 Agent 提供外部实时信息。

8. 价格爬虫
是什么：用程序抓网页里的价格信息。
项目里：用 requests + BeautifulSoup 抓什么值得买搜索页的标题、价格和链接。
面试说：爬虫提供定向价格线索，Tavily 负责广泛搜索，两者互补。

9. RAG
是什么：检索增强生成，先检索资料，再让模型回答。
项目里：检索历史购买、拦截和偏好信息，作为 LLM 上下文。
面试说：RAG 让 Agent 的建议结合用户历史，而不是只看当前一句话。

10. ChromaDB
是什么：向量数据库，用来存储和检索文本记忆。
项目里：保存历史消费、购买和拦截记录。
面试说：ChromaDB 在项目里承担长期记忆库的角色。

11. Jaccard 相似度
是什么：用交集 / 并集衡量两个文本集合相似度。
项目里：对 ChromaDB 召回的历史记录做重排。
面试说：Jaccard 重排让更相关的历史记忆排在前面。

12. FSM
是什么：有限状态机，把流程拆成固定阶段。
项目里：意图识别、实体抽取、RAG、搜索、爬虫、LLM 审计等阶段。
面试说：FSM 让 Agent 流程可控，避免所有请求直接进单个 LLM 调用。

13. Harness
是什么：Agent 执行控制层，管理 LLM 和工具调用。
项目里：限制最大步骤，处理工具失败，强制收敛最终答案。
面试说：FSM 管流程，Harness 管 LLM 和工具怎么执行、怎么收敛。

14. 意图识别
是什么：判断用户输入属于什么任务。
项目里：区分商品问价、直接记账、余额加回、撤销、帮助。
面试说：意图识别让不同任务走不同路径，提高稳定性。

15. 实体抽取
是什么：从自然语言里提取关键对象。
项目里：从“我想买雪碧”提取“雪碧”，从“花了3块”提取金额。
面试说：实体抽取把用户输入变成后续工具可用的参数。

16. 输入治理
是什么：先清洗和分类输入，避免所有请求都进复杂链路。
项目里：帮助、记账、撤销、问价走不同分支。
面试说：输入治理减少 LLM 滥用，也减少误判和延迟。

17. 规则优先
是什么：确定性任务优先用规则和代码处理。
项目里：记账、撤销、余额加回不交给 LLM，而是直接操作账本。
面试说：规则优先可以降低 LLM 幻觉风险。

18. LLM 审计
是什么：让模型综合信息判断是否建议购买。
项目里：LLM 作为消费审计官，结合预算、历史、搜索和价格线索输出建议。
面试说：LLM 负责综合判断，不是凭空聊天。

19. Prompt 约束
是什么：给模型设定角色、任务边界和输出格式。
项目里：要求模型作为消费审计官，并输出购买建议和 PRICE_DATA。
面试说：Prompt 约束让模型输出更符合业务需要。

20. PRICE_DATA
是什么：给程序解析的结构化价格块。
项目里：例如 {"item":"雪碧","estimated_price":3.5}，后端解析后用于展示和记账。
面试说：自然语言给用户看，PRICE_DATA 给后端解析。

21. 输出约束
是什么：不让模型随意输出，而是按固定格式返回。
项目里：要求附带 PRICE_DATA；解析失败时有兜底和价格纠偏。
面试说：输出约束让 LLM 结果可以被程序继续使用。

22. 价格纠偏
是什么：修正模型或搜索结果里的异常价格。
项目里：饮料类商品如果价格大于 20 元，会按单瓶价格纠偏。
面试说：避免把整箱饮料价格当成单瓶价格。

23. 直接记账
是什么：已经发生消费时直接写账本。
项目里：输入“买了雪碧花了3块”会扣余额并写流水。
面试说：直接记账是确定性任务，不需要 LLM 审计。

24. 预算扣减
是什么：消费后从余额里扣钱。
项目里：确认购入或直接记账后更新当前余额。
面试说：预算扣减让 Agent 输出真正落到账本状态。

25. 余额加回
是什么：退款或修正时把钱加回余额。
项目里：输入“加回来3块”会增加余额并记录流水。
面试说：余额加回用于退款和账本修正。

26. 撤销流水
是什么：取消上一笔账本记录并恢复余额。
项目里：输入“撤销上一条”或“加错了”触发。
面试说：撤销功能用于处理误记账。

27. 账本闭环
是什么：消费从建议、购买、扣款、修正到复盘都被管理。
项目里：确认购入、直接记账、撤销、余额加回、历史记录都打通。
面试说：账本闭环让系统不是只回答，而是能完成实际业务动作。

28. 钉钉通知
是什么：通过钉钉机器人 Webhook 推送消息。
项目里：提问、审计结果、扣款、撤销等会发到钉钉。
面试说：钉钉通知形成外部提醒闭环。

29. Webhook
是什么：外部系统给你的一个 HTTP 地址，用来接收事件消息。
项目里：向钉钉 Webhook POST 消息。
面试说：Webhook 让系统能主动把事件推送到钉钉。

30. ThreadPoolExecutor
是什么：Python 线程池，用于异步或并发任务。
项目里：异步发送钉钉通知，并行处理搜索、爬虫、记忆检索。
面试说：线程池避免外部 I/O 阻塞 Streamlit 前端。

31. 外部 I/O
是什么：访问网络、文件、数据库等可能较慢的操作。
项目里：搜索 API、价格爬虫、钉钉通知都是外部 I/O。
面试说：我给外部 I/O 加异步和超时，避免页面卡死。

32. JSON
是什么：轻量结构化数据格式。
项目里：保存用户画像、历史、账本，也解析 PRICE_DATA。
面试说：JSON 用于结构化存储和模型输出解析。

33. SQLite
是什么：轻量级本地数据库。
项目里：FastAPI 版本用它保存历史、账本、余额和拦截记录。
面试说：SQLite 适合 Demo 和本地持久化，正式多用户可换云数据库。

34. threading.Lock
是什么：线程锁，防止并发写文件冲突。
项目里：保护 JSON / 账本写入。
面试说：锁保证多个线程写数据时不会互相覆盖。

35. 历史持久化
是什么：数据刷新或重启后仍然保留。
项目里：保存用户画像、历史会话、账本流水和 Trace。
面试说：持久化让系统能长期记住消费状态。

36. Trace
是什么：一次请求的执行轨迹。
项目里：记录 intent_check、entity_extract、rag_retrieve、web_search、price_crawler、llm_audit 等阶段。
面试说：Trace 让 Agent 从黑盒变成可观察链路。

37. Agent Trace 日志
是什么：专门记录 Agent 每次运行过程的日志。
项目里：包含 trace_id、query、stages、errors、total_latency。
面试说：Trace 日志帮助定位失败阶段和统计耗时。

38. 测试 / 评测
是什么：用样本验证 Agent 是否稳定。
项目里：用自建消费样本测试意图分流、金额提取、账本操作和链路耗时。
面试说：评测不只看回答，还看每个阶段是否正常。

39. 回归样本
是什么：每次修改后反复跑的一组固定测试问题。
项目里：覆盖问价、记账、加回、撤销、帮助等场景。
面试说：回归样本保证核心功能不会被改坏。

40. 轻量检索
是什么：相比深度网页抓取更快的搜索方式。
项目里：把搜索从深度抓取改成轻量搜索。
面试说：轻量检索降低延迟，更适合交互式 Agent。

41. 超时降级
是什么：外部工具太慢时不一直等，改用兜底方案继续。
项目里：Web Search 超时后记录错误，并基于已有信息继续生成。
面试说：超时降级保证 Agent 不会因为外部搜索卡死。

42. 成功率 100%
面试说法：这是指固定复测样本内都能返回结果，不代表所有真实场景永远 100%。

43. 异常数 0
面试说法：这是指那批测试样本里没有程序报错或链路中断。

44. 51s 降到 8.75s
面试说法：之前深度搜索慢，后来改成轻量搜索并加超时降级，10 条复测样本平均链路耗时从约 51s 降到约 8.75s。

45. 主线程入队 1.47ms
面试说法：钉钉通知改成线程池异步发送后，主线程只提交任务，不等待外部 Webhook，因此不会明显阻塞页面。

最短项目介绍：
我做的是一个个人消费决策 Agent。它不是直接让 LLM 裸聊，而是先做意图分流和实体抽取，再结合 RAG 历史记忆、Tavily Web Search、价格爬虫和 LLM 审计，输出购买建议和结构化价格。系统还支持确认购入、直接记账、余额加回、撤销、钉钉通知和历史持久化。我用 Trace 记录每个阶段耗时，并用回归样本验证核心链路。
