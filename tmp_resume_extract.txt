省钱智探 Agent：消费风控与记账智能体 个人项目 2026.02 - 2026.06
项目开源地址：https://github.com/jerrychow43-debug/jerry-insight
项目网址：https://jerry-insight-2epkw2pl4pfcyxhtyypizl.streamlit.app/
概述：面向个人消费决策中“买前查价难、买后记账散、历史消费难复盘”的问题，基于 Python / Streamlit
独立开发消费风控 Agent，支持商品问价、实时比价、RAG 历史记忆、预算扣减、钉钉通知、直接记账、余额
加回、撤销流水与历史持久化，形成“问价-审计-扣款-通知-复盘”的端到端闭环。
 Agent 流程控制：自研 FSM / Harness 执行层，将消费审计拆分为意图识别、实体抽取、RAG 检索、W
eb Search、价格信息处理、LLM 审计和账本确认等阶段，避免所有请求直接进入单一 LLM 调用链路。
 输入治理与账本闭环：设计规则优先的输入分流链路，支持帮助问答、商品问价、直接记账、余额加回、撤
销上一笔等多类意图；基于 38 条自建消费场景回归样本验证，意图分流、金额提取和账本操作均通过测试。
 RAG 混合检索：基于 ChromaDB 记录历史购买、拦截和偏好信息，结合 Jaccard 相似度与重排策略，将
用户历史消费上下文、实时搜索结果和价格线索融合进审计输入，提升个性化判断能力。
 异步调度与持久化：使用 ThreadPoolExecutor 异步处理钉钉 Webhook，避免外部 I/O 阻塞 Streamlit
前端；模拟 10 个外部通知任务时，主线程入队耗时约 1.47ms；使用 JSON / SQLite 持久化用户画像、
历史会话和消费流水，并通过 threading.Lock 保护账本写入。
 Trace 可观测与性能优化：构建 Agent Trace 日志，记录意图识别、实体抽取、RAG、Web Search、LL
M 审计等阶段耗时和异常状态；基于 Trace 定位 Web Search 为主要瓶颈，将深度搜索与二次网页抓取
调整为轻量检索并加入超时降级，10 条复测样本中闭环成功率 100%、异常数 0，平均链路耗时由约 51s
降至约 8.75s。
